1. Login Functionality

Test Case 1.1: Valid Login

Description: Verify successful login with valid credentials.

Preconditions: User is on the login page.

Steps:

  • Enter valid username: standard_user.
  • Enter valid password: secret_sauce.
  • Click on the Login button.

Expected Result: User is redirected to the inventory page.

Test Case 1.2: Invalid Username

Description: Verify login fails with an invalid username.

Preconditions: User is on the login page.

Steps:

  • Enter invalid username: invalid_user.
  • Enter valid password: secret_sauce.
  • Click on the Login button.

Expected Result: Error message "Username and password do not match any user in this service" is displayed.

Test Case 1.3: Invalid Password

Description: Verify login fails with an invalid password.

Preconditions: User is on the login page.

Steps:

  • Enter valid username: standard_user.
  • Enter invalid password: invalid_password.
  • Click on the Login button.

Expected Result: Error message "Username and password do not match any user in this service" is displayed.

Test Case 1.4: Empty Fields

Description: Verify login fails with empty username and password fields.

Preconditions: User is on the login page.

Steps:

  • Leave the username field empty.
  • Leave the password field empty.
  • Click on the Login button.

Expected Result: Error message "Username is required" is displayed.

Test Case 1.5: Special Characters in Username and Password

Description: Verify login behavior with special characters in username and password.

Preconditions: User is on the login page.

Steps:

  • Enter special characters in username: !@#$%^&*().
  • Enter special characters in password: !@#$%^&*().
  • Click on the Login button.

Expected Result: Error message "Username and password do not match any user in this service" is displayed.

Expected Result: Error message indicating the session has expired. You can only access '/inventory.html' when you are logged in.
